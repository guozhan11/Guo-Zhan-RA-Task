研究助理(RA)筛选任务 — 招聘广告数据分析

感谢申请清华大学经济管理学院胡佳教授课题组的研究助理职位。请完成以下小型数据任务。
预计用时 3–4 小时。我们看重的是思路、代码质量和对数据的细致程度，而不是“标准答案”。

一、数据文件
1. ra_task_ads.csv — 612 条真实 51job 招聘广告(节选)。字段:
   id, 公司名称, 关联公司名称, 岗位, 岗位描述, 岗位标签, 待遇, 学历, 所在城市, 发布时间
2. ra_task_firms.csv — 5,463 家中国上市公司名单。字段:
   股票代码, 证券简称, 公司全称, 证监会行业
说明: 数据为真实原始数据，未做任何清洗。

二、任务
Part 1 — 数据清洗(必做)
1. 用 Python(或 SQL/DuckDB)读入 "ra_task_ads.csv"，报告行数。
2. 文本字段里有形如 <$&0006&$> 的格式乱码，请清理。
3. 数据中可能存在重复广告，请识别并说明你如何判断“重复”，去重后剩多少条。

Part 2 — 公司匹配(必做)
4. 将广告的 公司名称 与 "ra_task_firms.csv" 中的上市公司匹配，统计有多少条广告 / 多少家上市公司能匹配上。
5. 你会发现不少广告主是上市公司的子公司或分支机构，例如某保险公司的省分公司、某集团的物业子公司，其名称与母公司全称并不完全一致。请说明你的处理思路，并尽量把它们归并到对应的上市母公司。
6. 简要说明: 你的匹配可能产生哪些误匹配(false positive)? 举一个你在本数据中实际遇到或能预见的例子。

Part 3 — 文本编码: AI 含量(必做)
7. 给每条广告判断一个“AI / 数字化技术含量”的分数，可用 0/1 二分类，或 0–3 分级，自行定义并说明标准。
   可以用关键词词典法，或调用大语言模型 API（如有经验，这是加分项）。
8. 按 “发布时间”(年份)汇总，报告 AI 含量广告的占比随时间的变化，并简单可视化或列表。

Part 4 — 发现与反思(必做，简短)
9. 用 3–5 句话写出一个你从数据中看到的有意思的发现。
10. 再写出一个你认为会影响该发现可信度的数据问题或局限(data caveat)。

三、加分项(选做)
用 LLM API 完成 Part 3 并做信度检验，如人工核对若干条；把 Part 2 的匹配做得更稳健；合理的代码结构 / Git / 文档。

四、提交方式
- 一个压缩包或 GitHub 仓库链接，包含: 可运行的代码、简短说明(README 或 notebook，可中可英)，以及关键产出，包括匹配结果、年度占比表/图。
- 请说明你用到的工具、库，以及大致花费的时间。
- 发送至 hujia@sem.tsinghua.edu.cn，标题: “RA任务—姓名—学校”。

如有问题，数据本身的任何“看起来不对”的地方都可能是有意保留的，请按你的判断处理并说明即可。
